# 효과적인 프롬프트 패턴과 실전 활용 팁 (Dos and Don'ts)

Claude Code를 더욱 안전하고, 저렴하고, 똑똑하게 활용하기 위한 실전 노하우와 프롬프트 패턴을 소개합니다.

---

## 1. 🛡️ 플랜(Plan) 모드 필수 사용 (토큰 및 비용 최적화)

Claude Code에는 두 가지 작동 모드가 있습니다. `Shift + Tab`을 눌러 모드를 전환할 수 있습니다.

| 모드 | 설명 |
|---|---|
| **플랜(Plan)** | 코드를 즉시 수정하지 않고 작업 계획과 구조만 먼저 제시 |
| **에딧(Edit)** | 지시를 받자마자 실제로 파일들을 생성 및 수정 |

**💡 권장 워크플로우: 플랜 먼저, 에딧은 나중에!**
처음부터 에딧 모드를 쓰면 AI가 의도와 다르게 수많은 파일을 잘못 수정할 위험이 있습니다. 
1. **플랜 모드**에서 작업 계획을 먼저 설명받습니다.
2. 계획을 검토하고 문제가 없으면 승인(Accept)하여 **에딧 모드**로 전환해 실행합니다.

**플랜 모드를 써야만 하는 진짜 이유 (최적화)**
* **컨텍스트 윈도우 보존:** 에딧 모드로 바로 코드를 생성하면 여러 파일을 훑는 과정에서 컨텍스트가 꽉 차버립니다. 플랜 모드로 계획만 세운 뒤 승인하면, 훨씬 적은 컨텍스트로 깔끔하게 코드를 생성할 수 있습니다.
* **비용(토큰) 절감:** 잘못된 코드를 롤백하고 다시 짜라고 지시하는 과정 자체가 엄청난 토큰 낭비입니다. 처음엔 귀찮아도 플랜을 먼저 보는 습관이 결과적으로 비용을 아낍니다.

## 2. 💡 실전 코딩 꿀팁 및 프롬프트 패턴

### 🎙️ 음성 입력 활용 (Whisper Flow)
키보드로 길게 타이핑하는 대신 **Whisper Flow**를 이용해 말로 지시해 보세요. 
프롬프트는 구체적일수록 결과물의 퀄리티가 올라갑니다. 말로 지시하면 짧은 시간에 훨씬 더 풍부한 컨텍스트와 개발자의 의도를 AI에게 전달할 수 있어 매우 효과적입니다.

### 🖼️ 이미지/스크린샷 드래그 앤 드롭 적극 활용
프론트엔드 UI/UX 작업 시, 화면의 톤앤매너나 복잡한 레이아웃을 말로 길게 설명하는 것보다 **참고할 이미지를 터미널 창에 그대로 드래그 앤 드롭**하는 것이 100배 효과적입니다. 
Mermaid 형식의 아키텍처 다이어그램 등도 이미지로 던져주면 클로드가 훨씬 적은 토큰으로 의도를 정확히 파악합니다.

### 🌳 Git 연동 필수 (안전망 구축)
AI는 완벽하지 않습니다. 프로젝트가 커질수록 AI가 엉뚱한 방향으로 코드를 짰을 때 언제든 이전 상태로 돌아갈 수 있는 버전 관리가 필수입니다. Claude Code에게 복잡한 작업을 지시하기 전, 항상 현재 상태를 커밋해 두는 습관을 들이세요.
