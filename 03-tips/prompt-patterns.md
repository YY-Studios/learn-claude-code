# 기본 명령어 & 실전 팁

## 1. 플랜(Plan) 모드 vs 에딧(Edit) 모드

Claude Code에는 두 가지 모드가 있습니다.

| 모드 | 설명 |
|---|---|
| **플랜(Plan)** | 코드를 수정하지 않고 작업 계획만 설명 |
| **에딧(Edit)** | 실제로 파일을 수정 |

처음부터 에딧 모드를 쓰면 AI가 의도와 다르게 여러 파일을 한꺼번에 수정할 위험이 있습니다.

**권장 순서:**
1. 플랜 모드에서 작업 계획 확인
2. 계획 검토 후 문제 없으면 에딧 모드로 전환
3. 실행

> 💡 처음엔 귀찮아도 플랜 먼저 보는 습관이 중요합니다. 잘못된 코드를 되돌리는 게 훨씬 더 번거롭습니다.

---

## 2. 실전 꿀팁

### 음성 입력 활용 (Whisper Flow)
키보드로 길게 타이핑하는 대신 **Whisper Flow**로 말하면, 짧은 시간에 훨씬 풍부한 컨텍스트를 전달할 수 있습니다. 지시가 구체적일수록 결과물 퀄리티가 올라가기 때문에 음성 입력이 특히 효과적입니다.

### Git 연동 필수
프로젝트가 커질수록 AI가 잘못된 방향으로 코드를 짰을 때 되돌릴 수 있는 버전 관리가 필수입니다. Claude Code 사용 전에 Git 연동을 먼저 해두세요.

```bash
git init
git add .
git commit -m "before claude code"
```

### 터미널 알림 설정
긴 작업이나 여러 에이전트를 동시에 돌릴 때 터미널 알림을 연동해두면, 작업하다가도 완료 여부를 바로 파악할 수 있어 유용합니다.
