# 语音模式使用方法

Claude Code 支持语音输入模式。这是一个在不想打字时通过语音发出指令的实用功能。

---

## 激活语音模式

在终端中输入 `/voice` 命令即可激活语音输入模式。

```bash
/voice
```

激活后，终端底部会显示 🎤 麦克风图标。

---

## 使用方法

**基本流程：**

按住 Space 键开始说话。例如，可以说"在用户表中添加邮箱列"。说完后松开 Space 键，语音将转换为文字。此时不会自动发送，请确认转换结果，如有需要可进行修改，然后按 Enter 键发送。

**注意事项：**

松开按键后不会立即发送，而是转换为文字后需要手动按 Enter 发送。这意味着您有机会确认并修改转换结果。

---

## 中文设置

默认语言设置为英语。如需使用中文语音识别，请输入 `/config` 命令，然后在 `Language` 选项中选择 `Chinese`。

```bash
/config
```

### ⚠️ 中文输入法冲突问题

在中文 IME（输入法）开启的状态下向 `/config` 输入 `Chinese` 时，Enter 键可能会先被识别为确认中文字符，导致设置无法正确应用。

**解决方法：**

切换到英文输入模式（Ctrl + Space 或 Shift + Space）。在该状态下输入 `/config`，在 Language 选项中输入 Chinese 后按 Enter 确认，即可正常完成设置。

---

## 停用

再次输入相同的 `/voice` 命令，即可以切换方式停用语音模式。

```bash
/voice
```

---

## ✅ 适合使用的场景 vs ❌ 应避免的情况

**适合使用语音模式的场景：**

需要较长自然语言指令时非常有用。例如说明架构或传达需求等上下文较长的情况、进行快速问答时、想放开键盘边思考边说话时、一边看屏幕一边指示 UI 修改时都很适合。

**应避免使用语音模式的情况：**

需要输入精确代码片段时，直接打字更为合适。需要文件名、路径、API 密钥等精确字符串时，输入 JSON 或 YAML 等结构化数据时，需要粘贴终端错误日志时，直接输入比语音更为准确。

**推荐的混合使用方法：**

先用语音指示整体方向，再通过直接打字补充细节，这种方式效率最高。

---

## 故障排查

### 语音无法被识别时

请先检查麦克风权限（在操作系统设置中）。确认是否按住 Space 键足够长的时间，重新输入 `/voice` 重新激活，或尝试重启终端。

### 中文被识别为英文时

请进入 `/config` 命令，确认 Language 是否设置为 Chinese。如果已恢复为 English，请参考上方"中文输入法冲突问题"章节重新进行设置。

### 转换后的文字有误时

语音识别的准确性取决于发音、周围噪音和麦克风质量。转换后请务必在按 Enter 之前确认并修改。特别是部署或删除等重要操作，建议直接打字输入。

## 💡 实战使用技巧

### 与 Plan Mode 结合使用

用语音模式说"帮我做用户资料编辑功能，可以修改姓名、邮箱和头像"，然后按 Shift + Tab 查看计划。之后再通过打字对细节进行调整。

### 传达较长的需求

复杂的业务逻辑或多步骤流程，用语音描述反而可能更快。例如可以一次性说明："帮我做注册流程。顺序是：邮箱重复检查 → 发送 OTP → 验证码确认 → 设置密码 → 填写资料。使用 Supabase Auth，每个步骤都要加入验证。"

### 多任务处理

在进行代码审查或其他工作的同时，可以放开双手通过语音向 Claude 发出指令。无需使用键盘，自由工作的同时用语音发出命令，这正是其优势所在。
