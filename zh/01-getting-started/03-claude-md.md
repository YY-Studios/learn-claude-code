# 项目初始设置与上下文管理 (CLAUDE.md)

使用 AI 智能体时最需要注意的是**上下文溢出与幻觉（Hallucination）**问题。防止这些问题的核心工具就是 `claude.md` 文件。

---

## ⚠️ 核心：请务必在项目根目录（Root）下运行
Claude Code 以当前目录为基准搜索文件。如果在子文件夹（例如 `/apps/api`）中运行，将无法读取整个项目结构，导致生成错误代码或找不到文件的问题。请养成始终在最顶层根目录下运行的习惯。

---

## `/init` 命令与 `CLAUDE.md` 层级结构
在终端中运行 `/init`，会生成作为项目说明书的 `claude.md`。
- **全局应用：** `~/.claude/claude.md`（所有项目的通用规则。例如："始终用中文回答"）
- **项目应用：** `项目根目录/claude.md`（该项目专用的架构与规范）

---

## 💡 `claude.md` Token 优化与拆分策略（重要）
由于 AI 每次都会读取 `claude.md`，文件过长会大量消耗 Token。
* **保持在 300 字以内：** 根目录中的 `claude.md` 只保留核心摘要，尽量控制在 300 字以内（推荐做法）。
* **按文件夹拆分：** 将数据库相关规则放在 `/supabase/claude.md`，API 相关规则放在 `/api/claude.md`。这样 Claude 只在处理对应文件夹时才读取详细规则，效率大幅提升。

---

## 触发关键词（Trigger Keyword）设置
可以将重复的流水线（例如：测试通过后提交并推送）以触发关键词的形式注册到 `claude.md` 中。
> "输入'部署（Ship）'这个词时，按照 1. 运行测试 2. 失败则中止 3. 通过则 git add & commit 4. push 的顺序执行"
之后只需在 Claude Code 中输入"部署"，上述一系列流程就会自动执行。

---

## 🤝 复合工程（团队级上下文同步）
将 `claude.md` 提交到 Git 仓库，供整个团队共享。某位团队成员更新架构规则后，其他团队成员也能在相同的规则和上下文下使用 AI。

---

**💡 小贴士：不要手动修改，让 AI 来做**
在对话过程中确定了新的编码规范或规则后，无需手动打开 `claude.md` 文件进行编辑。
> "把我们刚才定好的模式添加到 claude.md 里"
只需这样指示，Claude 会自动更新文件。
