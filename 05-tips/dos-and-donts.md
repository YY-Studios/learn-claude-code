# 클로드 코드 잘 되는 것과 안 되는 것 (Dos and Don'ts)

## ✅ 적극 권장하는 것 (Dos)
* **Plan Mode 사용:** `Shift + Tab`으로 항상 계획부터 확인하고 코드를 수정하게 하세요.
* **TDD 기반 짧은 호흡:** 작은 변경 -> 린트/테스트 -> 커밋의 짧은 주기를 유지하세요.
* **스크린샷 드래그 앤 드롭:** UI 톤앤매너나 아키텍처 구조는 말보다 이미지로 던져주세요.
* **에러 로그 통째로 넘기기:** 본인의 주관적 해석을 빼고 터미널 에러 로그 전체를 복붙하세요.

## ❌ 절대 피해야 할 것 (Don'ts)
* **하나의 세션에서 여러 기능(피처) 개발하기:** 컨텍스트가 오염되어 곧 환각(Hallucination)이 발생합니다. 기능이 끝나면 과감히 `/clear` 하세요.
* **대용량 파일 직접 분석시키기:** 수만 줄의 JSON이나 텍스트를 대화창에서 파싱하게 하지 마세요. 파싱 스크립트를 짜달라고 한 뒤 로컬에서 돌리세요.
* **루트 `CLAUDE.md`에 모든 정보 때려 넣기:** API 스펙 전체를 넣지 말고, 별도 파일로 뺀 뒤 경로만 참조하게 하세요 (Lazy Loading).
