# 비용 최적화 & 토큰 관리

Claude Code는 사용한 토큰만큼 비용이 발생합니다. 어디서 토큰이 새는지 파악하고 줄이는 방법을 정리합니다.

---

## 현재 토큰 사용량 확인

```
/context
```

항목별로 분류된 토큰 사용량을 바(Bar) 형태로 보여줍니다.

| 항목 | 의미 |
|---|---|
| `System Prompt` | CLAUDE.md 등 시스템 지시사항 |
| `System Tools` | 연결된 MCP 서버, 도구 정의 |
| `User/Assistant` | 실제 대화 내용 |

---

## 토큰을 많이 먹는 패턴 TOP 5

### 1. 무거운 MCP를 항상 켜두기

Notion, Linear처럼 방대한 데이터를 다루는 MCP는 **숨만 쉬어도** `System Tools` 토큰을 10% 이상 잡아먹습니다.

```
해결: 쓸 때만 켜기
/mcp → 해당 서버 비활성화/활성화
```

### 2. CLAUDE.md가 너무 길어짐

루트 CLAUDE.md에 모든 내용을 넣으면 매 턴마다 수천 토큰이 낭비됩니다.

```
해결: 폴더별 CLAUDE.md로 분리
루트: 핵심 규칙만 50~100줄
서브폴더: 해당 도메인 규칙만 → 작업할 때만 로드
```

→ [CLAUDE.md 계층 구조](../02-context/claude-md-hierarchy.md) 상세 참고

### 3. 하나의 세션에서 여러 기능 개발

컨텍스트가 누적될수록 Claude가 처리해야 할 히스토리가 늘어나고, 점점 느려지며 비용도 오릅니다.

```
해결: 한 세션 = 한 피처 원칙
기능 완료 후 반드시 /clear
```

### 4. 대용량 파일을 대화창에서 직접 분석

수천 줄의 JSON, 로그 파일을 대화창에 붙여넣으면 한 번에 수만 토큰이 소비됩니다.

```
해결: 스크립트 오프로드
Claude에게 분석 스크립트를 짜달라고 한 뒤
로컬에서 실행 → 결과 요약만 Claude에게 전달
```

→ [스크립트 오프로드](../03-workflow/script-offload.md) 상세 참고

### 5. Subagent 남발

서브에이전트는 각자 독립 컨텍스트를 사용합니다. 3개를 병렬 실행하면 토큰이 최대 3배 소비됩니다.

```
해결: 작업이 진짜 독립적일 때만 병렬 실행
독립적이지 않다면 단일 세션이 훨씬 효율적
```

## 모델별 비용 전략

Claude Code는 작업 중에도 `/model`로 모델을 전환할 수 있습니다.

| 모델 | 상대 비용 | 적합한 작업 |
|---|---|---|
| Opus | 가장 높음 | 아키텍처 설계, 복잡한 버그, 중요한 결정 |
| Sonnet | 중간 | 일반 코딩, 병렬 서브에이전트 |
| Haiku | 가장 낮음 | 파일 탐색, 단순 리네이밍, 빠른 질문 |

**서브에이전트 모델 고정:**

```bash
# 메인 세션: Opus (판단), 서브에이전트: Sonnet (실행)
export CLAUDE_CODE_SUBAGENT_MODEL="claude-sonnet-4-5"
```

---

## 컨텍스트 관리 3단계

| 상황 | 명령어 | 효과 |
|---|---|---|
| 컨텍스트가 70% 이상 찼을 때 | `/compact` | 핵심 요약만 남기고 압축 |
| 기능 하나가 완료됐을 때 | `/clear` | 완전 초기화, 새 세션 시작 |
| 세션 간 맥락을 이어가야 할 때 | `TODO.md` 업데이트 후 `/clear` | 맥락은 파일로, 메모리는 비우기 |

## 비용 모니터링 체크리스트

작업 시작 전:
- [ ] `/context`로 현재 토큰 사용량 확인
- [ ] 사용하지 않는 MCP 서버 비활성화
- [ ] 작업 범위에 맞는 모델 선택

작업 중:
- [ ] 기능 완료마다 `/clear`
- [ ] 대용량 파일은 스크립트로 오프로드
- [ ] `think hard` 는 꼭 필요할 때만 (내부 추론 토큰 증가)

작업 후:
- [ ] 반복 패턴이 있으면 CLAUDE.md가 아닌 폴더 내 CLAUDE.md로 이동
- [ ] 자주 쓰는 작업은 커스텀 커맨드나 훅으로 자동화
