# 다른 AI와 협업 (Cross-AI Review)

Claude Code와 핑퐁을 거듭하며 제자리걸음을 할 때가 있습니다. 억지로 계속 싸우지 말고 **다른 AI의 관점**을 빌려오세요.

---

## 언제 써야 하나?

- 같은 버그를 3번 이상 고쳐도 재발하는 경우
- Claude가 점점 더 복잡한 방향으로 가는 느낌이 드는 경우
- 근본적인 아키텍처 결정에서 확신이 없는 경우
- 컨텍스트 늪에 빠져 응답 품질이 눈에 띄게 떨어진 경우

---

## `/export` 명령어

현재 세션의 대화 히스토리와 코드 컨텍스트를 파일로 추출합니다.

```bash
/export                    # 기본 파일로 내보내기
/export conversation.md    # 파일명 직접 지정
/export --clipboard        # 클립보드에 복사 (바로 붙여넣기 편함)
```

---

## Cross-AI Review 워크플로우

```
1. /export --clipboard 으로 현재 컨텍스트 복사
2. ChatGPT(o3), Gemini 2.0, Grok 등에 붙여넣기
3. 아래 프롬프트로 비평 요청
4. 힌트를 들고 Claude Code에 /clear 후 새 세션 시작
```

**다른 AI에게 요청하는 프롬프트:**
```
Claude Code와 이 문제를 풀고 있는데 막혔어.
Claude의 접근 방식에서 잘못된 점이 뭔지,
완전히 다른 해결책이 있는지 비평해줘.
Claude가 놓쳤을 가능성이 높은 부분을 중심으로.
```

---

## Claude 두 개로 서로 검토시키기

공식 권장 방법 중 하나로, 한 Claude가 코드를 작성하는 동안 다른 Claude가 검토하거나 테스트하게 하는 방식이 있습니다. 같은 문제를 완전히 다른 컨텍스트로 보게 해서 사각지대를 없애는 효과가 있어요.

```
터미널 창 1: 구현 담당 Claude Code
터미널 창 2: 리뷰 담당 Claude Code (또는 claude.ai 웹)

→ 창 1에서 작성한 코드를 창 2에 붙여넣고
  "이 코드의 문제점과 개선할 수 있는 부분을 비평해줘"
```

## AI별 강점 비교 (참고)

| AI | 잘 하는 것 |
|---|---|
| Claude (Opus) | 복잡한 추론, 아키텍처 설계, 긴 컨텍스트 |
| ChatGPT (o3) | 버그 찾기, 수학적 추론, 코드 리뷰 |
| Gemini 2.0 | 최신 정보, 멀티모달, 빠른 초안 |

> 💡 어떤 AI도 완벽하지 않습니다. 중요한 결정일수록 두 개 이상의 AI에게 검토받는 것이 좋습니다.
