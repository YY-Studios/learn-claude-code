## ⌨️ 클로드 코드 핵심 명령어(Command) 정리

클로드 코드의 효율적인 토큰 관리와 세션 관리를 위해 꼭 알아두어야 할 명령어들입니다.

### 대화창 내부 명령어 (슬래시 커맨드)

* `/init`: 프로젝트의 전반적인 세팅과 컨텍스트가 담긴 `claude.md` 파일을 생성합니다.
* `/clear`: AI와 나누었던 대화의 모든 컨텍스트를 초기화(리셋)하여 토큰을 비웁니다.
* `/compact`: 모든 기억을 지우는 대신, 대화 내용 중 중요한 정보만 압축하여 남기고 토큰 사용량을 줄여줍니다.
* `/rename`: 현재 진행 중인 세션에 'DB 스키마 업데이트'와 같이 이름을 지정하여 나중에 히스토리에서 쉽게 찾을 수 있도록 합니다.
* `/help`: 클로드 코드에서 사용할 수 있는 전체 명령어 목록과 단축키 도움말을 확인합니다.
* `/model`: 작업에 사용할 AI 모델(예: Sonnet, Haiku 등)을 변경합니다. 복잡한 로직 설계는 성능이 좋은 모델로, 단순한 오타 수정이나 린트(Lint) 검사는 빠르고 저렴한 모델로 변경해 비용을 최적화할 수 있습니다.
* `/cost`: 현재 세션에서 사용한 토큰 양과 누적 비용(USD)을 실시간으로 확인합니다.
* `/commit`: 작업 중인 파일들의 변경 사항(Diff)을 AI가 분석하여, 팀 컨벤션에 맞는 Git 커밋 메시지를 자동으로 작성하고 커밋을 수행합니다.
* `/mcp`: 외부 도구, 로컬 데이터베이스, 슬랙(Slack) 등을 클로드 코드와 직접 연동하기 위한 MCP(Model Context Protocol) 서버를 구성합니다.

### 터미널 명령어 (클로드 외부 실행)

* `claude --continue` (또는 `-c`): 직전에 끝마쳤던 클로드 세션을 다시 불러와 작업을 이어나갑니다.
* `claude --resume` (또는 `-r`): 지금까지 클로드와 나누었던 모든 대화 히스토리를 보여주고, 원하는 이전 세션으로 다시 들어갈 수 있게 해줍니다. 
* `claude "명령어"`: 대화창을 열고 타이핑할 필요 없이, 실행과 동시에 첫 번째 지시를 바로 전달합니다. (예: `claude "components 폴더의 버튼 컴포넌트에 다크모드 적용해줘"`)
* `claude -p "명령어"`: 대화형 모드(REPL)로 진입하지 않고, 명령어에 대한 결과만 터미널에 출력(Print)한 뒤 바로 종료합니다. 단발성 질문이나 파이프(`|`) 연산과 함께 스크립트처럼 쓰기 좋습니다.
* `claude update`: 클로드 코드 CLI를 최신 버전으로 업데이트합니다.
