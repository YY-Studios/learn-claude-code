# 핵심 명령어 및 단축키 (Quick Reference)

클로드 코드의 효율적인 토큰 관리와 작업 속도를 끌어올려 줄 필수 명령어 및 단축키 모음입니다.

---

## ⌨️ 꼭 알아야 할 핵심 단축키

| 단축키 | 동작 |
|---|---|
| `Shift + Tab` | **플랜(Plan) ↔ 에딧(Edit) 모드 전환** — 코드를 바로 수정하지 않고 계획부터 세울 때 필수 |
| `ESC` 1번 | 진행 중인 작업 즉시 중단 (Interrupt) |
| `ESC` 2번 연속 | **되감기(Rewind)** — 이전 프롬프트 상태로 되돌아갑니다 |

---

## ⚡ 터미널 내 Bash 명령어 실행 (`!`)

클로드 코드 안에서 깃 상태를 보거나 서버를 켤 때, 터미널을 새로 열 필요가 없습니다. 명령어 앞에 `!`를 붙이면 로컬 배시(Bash) 명령이 즉시 실행됩니다.

```
!git status        → 변경된 파일 확인
!git diff          → 변경 내용 상세 확인
!npm run dev       → 로컬 서버 실행
!npm run test      → 테스트 실행
```

## 💬 필수 슬래시 커맨드

### 세션 & 컨텍스트 관리

| 커맨드 | 설명 |
|---|---|
| `/init` | 프로젝트 설명서인 `CLAUDE.md`를 자동 생성합니다 (프로젝트 초기 세팅 시 필수) |
| `/clear` | 쌓여있는 대화 컨텍스트를 완전히 초기화합니다 |
| `/compact` | 대화 내역 중 핵심 요약만 남기고 컨텍스트를 압축해 토큰을 절약합니다 |
| `/context` | 현재 세션에서 사용 중인 토큰 양을 바(Bar) 형태로 시각적으로 확인합니다 |

> 💡 `/context`를 치면 `System Prompt`, `System Tools`, `User/Assistant` 항목별로 토큰 사용량이 나뉩니다. MCP를 연결했는데 `System Tools`가 많이 잡힌다면 비활성화를 고려하세요. → [MCP 토큰 관리](../04-agents/mcp.md)

### 메모리 & 설정

| 커맨드 | 설명 |
|---|---|
| `/memory` | 현재 로드된 메모리(`CLAUDE.md`, `MEMORY.md` 등)의 내용을 확인합니다 |
| `/status` | 현재 로그인된 계정, 구독 플랜, 연결된 MCP 서버 상태를 종합적으로 확인합니다 |
| `/allowed-tools` | AI에게 허용된 도구(파일 쓰기, 로컬 명령어 실행 등) 목록을 확인하고 권한을 관리합니다 |

### 모델 & 출력

| 커맨드 | 설명 |
|---|---|
| `/model` | 작업 난이도에 따라 모델을 전환해 토큰을 최적화합니다 |
| `/output-style` | AI의 응답 스타일을 변경합니다 |

**`/model` 선택 기준:**

| 모델 | 적합한 작업 |
|---|---|
| `Opus` | 복잡한 아키텍처 설계, 원인 불명 버그 해결, 중요한 의사결정 |
| `Sonnet` | 일반적인 코딩 작업, 병렬 서브에이전트에게 맡길 때 (기본값) |
| `Haiku` | 파일 탐색, 간단한 리네이밍, 빠른 질문 |

**`/output-style` 선택지:**

| 스타일 | 설명 | 추천 상황 |
|---|---|---|
| `auto` | 상황에 맞게 자동 조절 (기본값) | 일반 사용 |
| `concise` | 핵심만 짧게 | 빠른 반복 작업, 코드 수정 |
| `verbose` | 상세한 설명 포함 | 학습 목적, 처음 보는 코드베이스 |
| `explanatory` | 코드 변경 이유를 함께 설명 | 코딩 학습, 코드 리뷰 |
| `warning` | 위험한 작업 시 경고를 강하게 표시 | 프로덕션 작업 시 안전망 |

### 에이전트 & 자동화

| 커맨드 | 설명 |
|---|---|
| `/agents` | 현재 프로젝트에 정의된 서브에이전트 목록을 확인하고 관리합니다 |
| `/mcp` | 연결된 MCP 서버 목록을 확인하고 활성화/비활성화합니다 |

### 내보내기

| 커맨드 | 설명 |
|---|---|
| `/export` | 현재 세션 컨텍스트를 다른 챗봇(ChatGPT, Gemini 등)으로 가져갈 수 있도록 추출합니다 |
| `/export --clipboard` | 클립보드에 직접 복사 |

## 🛠️ 나만의 커스텀 슬래시 명령어 만들기

자주 쓰는 파이프라인을 슬래시 명령어로 모듈화할 수 있습니다.

1. 프로젝트 내 `.claude/commands/` 폴더를 생성합니다.
2. `ship.md` 등 원하는 이름으로 파일을 만들고 실행할 워크플로우를 작성합니다.
3. Claude Code 재시작 후 `/ship`을 입력하면 해당 파이프라인이 자동 실행됩니다.

→ 자세한 내용은 [커스텀 슬래시 명령어](../04-agents/custom-commands.md) 참고
