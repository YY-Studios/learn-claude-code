# 他のAIとの協業 (Cross-AI Review)

Claude Codeとピンポンを繰り返して、同じところで止まってしまうことがあります。無理に戦い続けるのをやめて、**他のAIの視点**を借りましょう。

---

## いつ使うべきか？

- 同じバグを3回以上修正しても再発する場合
- Claudeがどんどん複雑な方向に進んでいる気がする場合
- 根本的なアーキテクチャの決定に確信が持てない場合
- コンテキストの沼にはまって、応答品質が目に見えて落ちている場合

---

## `/export` コマンド

現在のセッションの会話履歴とコードコンテキストをファイルとして抽出します。

```bash
/export                    # 기본 파일로 내보내기
/export conversation.md    # 파일명 직접 지정
/export --clipboard        # 클립보드에 복사 (바로 붙여넣기 편함)
```

---

## Cross-AI Review ワークフロー

```
1. /export --clipboard 으로 현재 컨텍스트 복사
2. ChatGPT(o3), Gemini 2.0, Grok 등에 붙여넣기
3. 아래 프롬프트로 비평 요청
4. 힌트를 들고 Claude Code에 /clear 후 새 세션 시작
```

**他のAIに依頼するプロンプト:**
```
Claude Code와 이 문제를 풀고 있는데 막혔어.
Claude의 접근 방식에서 잘못된 점이 뭔지,
완전히 다른 해결책이 있는지 비평해줘.
Claude가 놓쳤을 가능성이 높은 부분을 중심으로.
```

---

## Claudeを2つ使って互いにレビューさせる

公式推奨の方法の一つとして、一方のClaudeがコードを書いている間に、もう一方のClaudeがレビューやテストを行う方式があります。同じ問題をまったく異なるコンテキストで見ることで、盲点をなくす効果があります。

```
터미널 창 1: 구현 담당 Claude Code
터미널 창 2: 리뷰 담당 Claude Code (또는 claude.ai 웹)

→ 창 1에서 작성한 코드를 창 2에 붙여넣고
  "이 코드의 문제점과 개선할 수 있는 부분을 비평해줘"
```

## AI別の強み比較 (参考)

| AI | 得意なこと |
|---|---|
| Claude (Opus) | 複雑な推論、アーキテクチャ設計、長いコンテキスト |
| ChatGPT (o3) | バグ発見、数学的推論、コードレビュー |
| Gemini 2.0 | 最新情報、マルチモーダル、素早いドラフト |

> 💡 どのAIも完璧ではありません。重要な決定ほど、2つ以上のAIにレビューしてもらうことをお勧めします。
