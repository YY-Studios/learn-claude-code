# カスタムスラッシュコマンドの作り方 (.claude/commands)

毎回繰り返すテスト・フォーマット・gitコミットの工程を、自分だけのスラッシュコマンド（Slash Command）としてモジュール化し、エージェントパイプラインを構築できます。

## 🚀 作成方法
1. プロジェクトのルートディレクトリに `.claude/commands/` フォルダを作成します。
2. 実行したいコマンド名でマークダウンファイルを作成します。（例: `ship.md`）
3. ファイルの中に実行するワークフローをプロンプト形式で記述します。

**`ship.md` の記述例:**
```md
# Description
テスト実行後、成功した場合にコミットしてプッシュします。

# Workflow
1. `npm run test`を実行する。
2. テスト失敗時はエラーログを出力し、即座に作業を中断する。
3. テスト通過時は`git add .`を実行する。
4. 変更内容を分析し、適切なコミットメッセージで`git commit`を行う。
5. `git push`を実行してリモートリポジトリにデプロイする。
```
Claude Codeを再起動した後、チャット画面に /ship と入力すると、上記のパイプラインが全自動で実行されます。
