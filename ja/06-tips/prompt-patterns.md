# 効果的なプロンプトパターンと実践活用のヒント（Dos and Don'ts）

Claude Code をより安全に、低コストで、賢く活用するための実践的なノウハウとプロンプトパターンを紹介します。

---

## 1. 🛡️ プラン（Plan）モードの必須活用（トークン・コスト最適化）

Claude Code には2つの動作モードがあります。`Shift + Tab` を押してモードを切り替えられます。

| モード | 説明 |
|---|---|
| **プラン（Plan）** | コードをすぐに修正せず、作業計画と構造だけを先に提示する |
| **エディット（Edit）** | 指示を受けた直後に実際にファイルを生成・修正する |

**💡 推奨ワークフロー：まずプラン、エディットは後で！**
最初からエディットモードを使うと、AIが意図と異なる形で大量のファイルを誤って修正するリスクがあります。
1. **プランモード**で作業計画をまず説明してもらいます。
2. 計画を確認して問題がなければ承認（Accept）し、**エディットモード**に切り替えて実行します。

**プランモードを使うべき本当の理由（最適化）**
* **コンテキストウィンドウの保全：** エディットモードで直接コードを生成すると、複数ファイルを参照する過程でコンテキストがすぐに埋まってしまいます。プランモードで計画だけ立ててから承認すると、はるかに少ないコンテキストでクリーンなコードを生成できます。
* **コスト（トークン）の削減：** 誤ったコードをロールバックして書き直しを指示するプロセス自体が膨大なトークンの無駄遣いです。最初は面倒でも、先にプランを確認する習慣がコストの節約につながります。

---

## 2. 💡 実践コーディングのヒントとプロンプトパターン

### 🧠 深い推論を引き出す（Think Hard / Think Harder）
単純なコード生成ではなく、複雑なアーキテクチャの設計や原因不明のバグを追う際に、プロンプトの先頭に `think hard` または `think harder` を付けてみてください。
> 例：「think harder. 現在のログインロジックで発生している無限再レンダリング問題の原因を分析し、解決策を3つのオプションで提示してください。」

これは公式に推奨されているプロンプトパターンで、AIがすぐにコードを出力する前に、より深く多角的な内部推論プロセスを経るよう誘導し、アウトプットの品質を大幅に向上させます。

### 🎙️ 音声入力の活用（Whisper Flow）
キーボードで長く入力する代わりに、**Whisper Flow** を使って音声で指示してみてください。
プロンプトは具体的であるほど出力の品質が上がります。音声で指示することで、短時間でより豊富なコンテキストと開発者の意図をAIに伝えられるため、非常に効果的です。

### 🖼️ 画像・スクリーンショットのドラッグ＆ドロップを積極的に活用
フロントエンドのUI/UX作業では、画面のトーン＆マナーや複雑なレイアウトを言葉で長々と説明するより、**参考画像をターミナルウィンドウにそのままドラッグ＆ドロップ**する方が100倍効果的です。
Mermaid形式のアーキテクチャ図などを画像で渡すと、Claudeがはるかに少ないトークンで意図を正確に把握します。

### 🌳 Git連携は必須（安全網の構築）
AIは完璧ではありません。プロジェクトが大きくなるほど、AIが意図しない方向でコードを書いた際にいつでも以前の状態に戻せるバージョン管理が必須です。Claude Code に複雑な作業を指示する前に、常に現在の状態をコミットしておく習慣を身に付けましょう。
