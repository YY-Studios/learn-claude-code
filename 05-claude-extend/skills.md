# Skills (AI에게 주는 업무 매뉴얼)

스킬은 일회성 프롬프트와 달리, **반복 작업을 자동화하고 일관된 품질을 얻기 위해 AI에게 제공하는 재사용 가능한 업무 매뉴얼**입니다.

---

## 왜 스킬을 써야 하나?

| 이유 | 설명 |
|---|---|
| **반복 자동화** | `/ppt`처럼 짧은 키워드 하나로 복잡한 작업 자동 실행 |
| **품질 일관성** | 정해진 순서·형식·체크리스트를 강제해서 항상 같은 결과물 |
| **팀 공유** | Git으로 공유하면 팀 전체가 동일한 자동화 시스템 사용 |

---

## CLAUDE.md vs 커스텀 커맨드 vs Skills

| | CLAUDE.md | 커스텀 커맨드 | **Skills** |
|---|---|---|---|
| 로드 시점 | 항상 전체 로드 | 호출 시 로드 | **필요할 때만 로드** |
| 컨텍스트 비용 | 높음 | 중간 | **매우 낮음** |
| 용도 | 전역 규칙 | 반복 워크플로우 | 도메인 전문 지식 |

---

## 2단계 로딩 방식 (핵심)

스킬이 컨텍스트를 절약하는 이유가 바로 이 2단계 로딩 덕분입니다.

```
1단계 (평상시)
  → 스킬 이름 + 한 줄 설명만 로드 (약 50~100바이트)
  → 컨텍스트 거의 차지 안 함

2단계 (호출 시)
  → 요청이 스킬 목적과 일치한다고 판단될 때만
  → SKILL.md 본문 + 참고 파일 전체 로드 (On-demand)
```

CLAUDE.md나 MCP 도구는 항상 전체 내용을 로드하지만, 스킬은 필요할 때만 로드되어 컨텍스트를 획기적으로 절약합니다.

---

## 스킬 저장 위치

```
~/.claude/skills/          # 개인용 (모든 프로젝트에서 사용)
.claude/skills/            # 프로젝트용 (해당 프로젝트에서만 사용)
```

팀과 공유하려면 `.claude/skills/`에 넣고 Git에 커밋하세요.

---

## 스킬 만들기

### 방법 1 — 플러그인으로 자동 생성 (추천)

```
/install-plugin skill-creator
```

설치 후 자연어로 요청하면 모범 사례가 적용된 SKILL.md를 자동 생성합니다.

```
"PPT 발표자료 자동 생성하는 스킬 만들어줘"
```

### 방법 2 — 직접 작성

`.claude/skills/my-skill/SKILL.md` 파일을 만들고 아래 구조로 작성합니다.

```markdown
---
name: ppt-generator
description: PPT 발표자료 자동 생성. 'PPT 만들어줘', '슬라이드 만들어줘', 
             '발표자료 필요해' 요청 시 트리거. 마크다운을 슬라이드 구조로 변환.
---

# PPT 생성 스킬

## 실행 순서
1. 주제와 대상 청중 파악
2. 목차 초안 작성 후 확인
3. 각 슬라이드 내용 작성
4. 핵심 메시지 강조 포인트 추가
```

---

## Description 작성이 핵심

스킬이 제대로 트리거되려면 description을 잘 써야 합니다.

```markdown
❌ 나쁜 예시
description: 문서 생성 스킬

✅ 좋은 예시  
description: PPT 발표자료 자동 생성. 'PPT 만들어줘', '슬라이드 필요해', 
             '발표자료 만들어줘' 요청 시 트리거. 목차 작성 → 내용 채우기 순서로 진행.
```

**핵심 기능 + 사용자가 쓸 법한 트리거 표현 3개 이상** 포함하는 게 좋습니다.

## 실전 활용 예시

```
/client-report   # 클라이언트 보고서 자동 생성
/code-review     # 코드 리뷰 체크리스트 실행
/ppt             # PPT 발표자료 생성
/deploy-check    # 배포 전 체크리스트 실행
```

> 💡 커스텀 커맨드(`.claude/commands/`)와 헷갈릴 수 있어요.  
> 커맨드는 정해진 워크플로우 실행, 스킬은 도메인 전문 지식을 AI에게 주입하는 용도입니다.
