# MCP 연동 및 토큰 관리 전략

MCP(Model Context Protocol)는 클로드 코드에 날개를 달아주는 외부 도구 연동 시스템입니다. 로컬 DB, 슬랙, 노션, 깃허브 등과 클로드를 직접 연결할 수 있습니다.

## ⚠️ 무거운 범용 MCP 주의사항
하지만 노션이나 리니어(Linear)처럼 방대한 데이터를 다루는 범용 MCP를 무작정 연결해 두면, **숨만 쉬어도(아무 질문을 하지 않아도) 시스템 프롬프트 토큰 용량을 엄청나게 차지**합니다. (전체 컨텍스트 윈도우의 10% 이상을 점유하기도 함)

## 💡 실전 최적화 팁
1. **정기적인 모니터링:** `/context` 명령어를 쳐서 `System Tools`가 토큰을 얼마나 먹고 있는지 수시로 확인하세요.
2. **사용하지 않을 땐 끄기:** 지금 당장 노션 문서 검색이 필요 없다면 `/mcp` 명령어 창에 들어가 해당 서버를 비활성화하세요.
3. **커스텀 래핑(Custom Wrapping):** 방대한 범용 MCP 대신, 내 프로젝트의 특정 기능(예: 특정 DB 테이블만 조회)만 수행하는 아주 가벼운 커스텀 MCP를 만들어 쓰는 것이 토큰 관리에 훨씬 유리합니다.
