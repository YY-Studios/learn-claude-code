# 무거운 작업 스크립트 오프로드 (Offload)

대용량 JSON 파싱, 수백 개 파일 일괄 변환, 복잡한 DB 데이터 추출 등을 대화창에서 직접 처리하게 하면 컨텍스트 토큰이 순식간에 고갈됩니다.

---

## 핵심 원칙

> **Claude에게 무거운 연산을 시키지 말고, 연산을 수행할 스크립트 작성만 시키세요.**

```
❌ Bad  → Claude가 직접 처리 (토큰 폭발)
✅ Good → Claude가 스크립트 작성 → 로컬 실행 → 결과 요약만 전달
```

---

## 실전 예시

### 대용량 JSON 분석

```
❌ "data.json 파일 안의 1만 줄 데이터를 분석해서 카테고리별로 분류해줘."

✅ "data.json을 읽어서 카테고리별로 분류하고 결과를 요약 출력하는 
   Node.js 스크립트를 짜줘."
```
→ 스크립트 로컬 실행 → 터미널 결과 복붙
→ "이 결과를 바탕으로 다음 UI 작업을 진행해."

---

### 파일 일괄 변환

```
❌ "src 폴더 안의 .js 파일 200개를 전부 TypeScript로 변환해줘."

✅ "src 폴더의 .js 파일을 TypeScript로 일괄 변환하는 
   bash 스크립트를 짜줘. 변환 결과를 파일별로 로그 출력할 것."
```

---

### Supabase 데이터 추출

```
❌ "users 테이블에서 지난 30일 비활성 유저를 추출해서 분석해줘."

✅ "지난 30일 비활성 유저를 추출해서 CSV로 저장하는 
   Supabase 쿼리 스크립트를 짜줘."
```

---

## 오프로드 판단 기준

아래 중 하나라도 해당되면 스크립트로 오프로드하세요.

- 파일 내용을 대화창에 직접 붙여넣어야 하는 경우
- 반복 처리 대상이 10개 이상인 경우
- 실행 결과가 100줄 이상 예상되는 경우
- DB에서 데이터를 직접 읽어 분석해야 하는 경우

---

## 결과 요약 전달 방법

스크립트 실행 후 결과가 길다면 요약해서 전달하는 것도 Claude에게 맡길 수 있습니다.

```
"방금 스크립트 실행 결과야. [결과 붙여넣기]
 이 중 에러가 발생한 항목만 추려서 다음 작업을 진행해줘."
```

> 💡 결과 전체를 붙여넣기보다, Claude에게 "결과에서 핵심만 뽑아달라"고 먼저 요청하면 토큰을 아낄 수 있어요.
