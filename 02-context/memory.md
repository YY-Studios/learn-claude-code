# 자동 메모리 시스템 (/memory)

> 자동 메모리는 **Claude Code v2.1.59 이상**에서 지원됩니다. `claude --version`으로 확인하세요.

---

## CLAUDE.md vs 자동 메모리 — 뭐가 다른가?

| | **CLAUDE.md** | **자동 메모리 (MEMORY.md)** |
|---|---|---|
| 작성 주체 | 사람이 직접 작성 | Claude가 스스로 작성 |
| 역할 | 규칙·컨벤션 지침서 | 세션에서 학습한 내용 기록 |
| 팀 공유 | Git에 커밋해서 공유 가능 | 개인 로컬에 저장 (공유 안 됨) |
| 저장 위치 | 프로젝트 루트 | `~/.claude/projects/<project>/memory/` |

> ⚠️ **원본 내용 수정:** 자동 메모리는 `MEMORY.md`로 관리되며, `CLAUDE.md`와는 완전히 별개 파일입니다. 팀원과 공유되는 게 아니라 **로컬에만 저장되는 개인 지식 저장소**입니다.

---

## 자동 메모리에 저장되는 것

Claude는 작업할 때 자신을 위해 노트를 저장합니다. 빌드 명령, 디버깅 인사이트, 아키텍처 노트, 코드 스타일 선호도 및 워크플로우 습관이 포함됩니다. 모든 세션마다 저장하지는 않으며, 향후 대화에서 유용할지 여부에 따라 기억할 가치가 있는지 결정합니다.

**저장되는 것 ✅**
- "이 프로젝트에서는 npm 대신 bun을 써" → 다음 세션부터 자동 반영
- API 호출 시 content-type 필수라는 디버깅 교훈
- 아키텍처 결정사항, 자주 빠지는 함정과 해결법

**저장 안 되는 것 ❌**
- "방금 작업 중인 파일 경로"
- "현재 디버깅 중인 에러 메시지"
- 세션에서만 의미 있는 일시적 정보

---

## 저장 파일 구조

```
~/.claude/projects/<project>/memory/
├── MEMORY.md          ← 인덱스 파일, 매 세션 자동 로드 (첫 200줄)
├── debugging.md       ← 디버깅 패턴 상세 (필요할 때만 로드)
├── patterns.md        ← 코드 패턴과 컨벤션
└── api-conventions.md ← API 설계 관련 메모
```

> 같은 Git 저장소 내 모든 워크트리와 하위 디렉터리는 **하나의 메모리 디렉터리를 공유**합니다.

---

## /memory 명령어 사용법

```
/memory
```

현재 로드된 메모리 파일 목록 확인 및 자동 메모리 토글 ON/OFF 가능합니다.

---

## 명시적으로 기억시키기

Claude에게 직접 말하면 자동 메모리에 저장됩니다.

```
"이 프론트엔드 컨벤션을 기억해둬"
"항상 pnpm을 사용한다는 걸 메모리에 저장해줘"
```

CLAUDE.md에 넣고 싶다면 명확하게 구분해서 말해야 해요.

```
"이 내용을 CLAUDE.md에 추가해줘"  ← CLAUDE.md에 저장
"이걸 기억해줘"                    ← 자동 메모리에 저장
```

---

## 실무 팁

- **주기적으로 메모리 정리 필요:** 패키지 매니저를 npm → bun으로 바꿨는데 예전 메모리에 npm이 남아있으면 혼란을 줍니다. `"이 메모리 지워줘"`, `"빌드 관련 내용 업데이트해줘"` 처럼 직접 요청하면 Claude가 파일을 수정합니다.
- **자동 메모리 비활성화:** 원하지 않으면 끌 수 있습니다.
  ```bash
  CLAUDE_CODE_DISABLE_AUTO_MEMORY=1
  ```
